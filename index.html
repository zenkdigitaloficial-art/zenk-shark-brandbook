Brandbook oficial

ZENK / Shark OS

Identidade visual, produto, UI e linguagem para construir uma operação comercial mais clara, rápida e brutalmente executável.

Assessoria que opera

A ZENK não vende fumaça de marketing. Ela organiza aquisição, vendas, rotina e crescimento.

Produto com cadência

Shark OS é cockpit, CRM, sprint, ligação e gestão diária. Sem firula. Sem teatro.

Motivação adulta

Gamificação existe para aumentar execução, não para fantasiar vendedor de personagem de arcade.

Essência

Quem somos e como decidimos

A marca precisa ser entendida em segundos por IA, designer e dev. O resto é decoração querendo parecer estratégia.

Missão

Construir máquinas comerciais previsíveis.

Unir marketing, vendas e operação para empresas venderem mais com processo, cadência, dados e execução diária.

Promessa

Menos achismo. Mais pipeline.

A ZENK transforma oportunidade em rotina comercial: lead certo, abordagem certa, CRM limpo, sprint claro e cobrança de resultado.

Personalidade

Direta, energética, humana.

  • Fala simples e firme.
  • Não enrola para parecer sofisticada.
  • Trata execução como prioridade, não como detalhe.
Princípios de produto

Operação antes de ornamento.

  • Todo card precisa ajudar uma decisão.
  • Todo alerta precisa indicar próxima ação.
  • Todo dado precisa ter dono, prazo e consequência.

Logo / mark

Placeholder oficial substituível

O símbolo abaixo é um placeholder elegante inspirado em Z + movimento/shark. Quando o logo final existir, substitua mantendo proporção, respiro e uso do azul.

Regras de uso

  • Não achatar, distorcer ou aplicar efeitos neon.
  • Manter área de respiro mínima igual a 25% da largura do mark.
  • Preferir azul sólido, branco sobre azul ou versão monocromática discreta.
  • Não usar tubarão literal agressivo. A inspiração é velocidade e foco, não mascote infantil.

UI Copy

Linguagem direta, útil e adulta

Copy do Shark OS deve empurrar ação. Nada de “opsie”, “uhulzinho” ou corporativês anestesiado. O vendedor não precisa de carinho genérico; precisa de clareza.

Labels

Curto e inequívoco.

Use “Próxima ação”, “Decisor”, “Atendidas”, “MRR”, “Ritmo”, “Dívida do dia”.

Alertas

Problema + ação.

Não diga apenas que algo está errado. Diga o que fazer agora.

Gamificação

Motivar sem infantilizar

Gamificação no Shark OS é reforço de comportamento comercial: ligação, resposta, reunião, venda e cadência. Não é carnaval visual.

XP

Experiência

Ganho por atividades que indicam execução: atendidas, reuniões confirmadas, follow-up no prazo, venda registrada.

Moedas

Recompensa

Usadas futuramente em avatar, loja e benefícios internos. Sempre com dourado discreto.

Patentes

Progressão

Exemplo: Rookie, SDR I, SDR II, Closer I, Closer II, Captain. Nomes sóbrios, sem fantasia exagerada.

Temporada

Mensal

Ranking e conquistas reiniciam por mês. O histórico permanece para evolução e reconhecimento.

Regras de estética

  • Dourado aparece em XP, moedas, conquistas e patentes.
  • Avatar é futuro, mas precisa ser opcional e secundário.
  • Ranking não deve humilhar. Deve direcionar comportamento.
  • Desafios entre workspaces devem comparar ritmo e evolução, não só volume bruto.
  • Sem neon, sem roxo genérico de IA, sem estética arcade.
Do / Don’t

O que fazer e o que nunca fazer

Essas regras evitam que o Shark OS vire mais um SaaS genérico com cor bonita e alma nenhuma.

Do

  • Use off-white como base e cards brancos para superfície.
  • Use azul ZENK para ações primárias e navegação.
  • Mostre próxima ação sempre que houver lead, tarefa ou alerta.
  • Use densidade operacional: informação útil sem poluição.
  • Use Syne para personalidade e Inter para clareza.
  • Conecte sprint, CRM, ligação e gamificação por dados.

Don’t

  • Não usar preto/laranja antigo como identidade principal.
  • Não criar visual gamer exagerado, neon ou roxo IA genérico.
  • Não usar dark mode como visual principal.
  • Não usar Manrope.
  • Não criar landing page com hero inútil para explicar produto operacional.
  • Não esconder erro com mensagem vaga.
IA / Dev

Como usar este brandbook

Use este arquivo como fonte fixa em prompts, specs, issues e implementação. Se a entrega contradiz este brandbook, a entrega está errada.

Prompt base para IA

# Contexto obrigatório Crie interfaces para ZENK / Shark OS seguindo o brandbook oficial. Priorize SaaS/CRM premium, claro, operacional e rápido. Use Syne para headings, Inter para UI, JetBrains Mono apenas para dados. Use azul ZENK como primário; verde sucesso; vermelho risco; dourado recompensa. Não use Manrope, neon, roxo IA genérico, dark mode principal ou estética gamer. # Regra de produto Toda tela deve deixar clara a próxima ação. CRM, sprint, ligação e gamificação devem estar conectados por dados.

Checklist antes de implementar

Visual

Base clara, azul ZENK, bordas sutis, sombras leves, sem estética antiga.

Produto

A tela mostra o que fazer agora? Se não mostra, ainda não está pronta.

Dados

Métrica tem contexto, ritmo, dono, prazo ou consequência?

Copy

A mensagem é direta, específica e acionável?

CRM

Lead tem decisor, status, próxima ação e histórico?

Gamificação

Recompensa comportamento comercial real sem infantilizar?

Regra final

Shark OS não é uma tela bonita para parecer moderno. É uma ferramenta para aumentar execução comercial. Se uma decisão visual não melhora clareza, velocidade ou cadência, corte. Bonito e inútil ainda é inútil — só que com sombra suave.
